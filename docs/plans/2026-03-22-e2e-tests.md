# E2E Tests Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Create comprehensive E2E tests covering all J-Obs API endpoints and functional areas.

**Architecture:** `@SpringBootTest(webEnvironment = RANDOM_PORT)` with `TestRestTemplate` for HTTP testing. Each functional area is an independent test class sharing a common base configuration. Tests boot a full Spring context with J-Obs auto-configured.

**Tech Stack:** JUnit 5, AssertJ, Spring Boot Test, TestRestTemplate, Spring WebSocket Client

---

## Test Structure

```
j-obs-spring-boot-starter/src/test/java/io/github/jobs/spring/e2e/
├── E2ETestBase.java              # Shared config, helpers, test app
├── DashboardPagesE2ETest.java    # HTML pages: /, /dashboard, /requirements, /login, /api-docs
├── RequirementsApiE2ETest.java   # GET/POST /api/requirements, /api/status, /api/capabilities
├── LogsApiE2ETest.java           # GET /api/logs, /api/logs/stats, /api/logs/levels
├── TracesApiE2ETest.java         # GET /api/traces, /api/traces/{id}, /api/traces/stats
├── MetricsApiE2ETest.java        # GET /api/metrics, /api/metrics/stats, /api/metrics/quick-stats
├── HealthApiE2ETest.java         # GET /api/health, /api/health/components, /api/health/summary
├── AlertsApiE2ETest.java         # CRUD /api/alerts, /api/alert-events, /api/alert-providers
├── SloApiE2ETest.java            # CRUD /api/slos, evaluate, history, summary
├── ProfilingApiE2ETest.java      # /api/profiling/cpu, /memory, /threads, /sessions
├── SqlAnalyzerApiE2ETest.java    # /api/sql/analyze, /issues, /slow-queries, /top-queries
├── ServiceMapApiE2ETest.java     # /api/service-map, /nodes, /connections
├── AuthenticationE2ETest.java    # Login, logout, session, API key, CSRF, rate limiting
└── WebSocketE2ETest.java         # WebSocket log streaming
```

## E2ETestBase Configuration

- `@SpringBootTest(webEnvironment = RANDOM_PORT)`
- `@TestPropertySource` with:
  - `j-obs.enabled=true`
  - `j-obs.security.enabled=false` (except AuthenticationE2ETest)
  - `j-obs.rate-limiting.enabled=false` (except rate limit tests)
  - All features enabled (logs, traces, metrics, health, alerts, profiling, slos, sql-analyzer, anomaly-detection, service-map)
- Helper methods: `url(path)`, `getJson(path)`, `postJson(path, body)`

## Tasks

### Task 1: E2ETestBase + DashboardPagesE2ETest + RequirementsApiE2ETest
Foundation: base class, HTML page tests, requirements/status/capabilities API tests.

### Task 2: LogsApiE2ETest + TracesApiE2ETest
Log querying, filtering, stats. Trace querying, detail, spans, stats.

### Task 3: MetricsApiE2ETest + HealthApiE2ETest
Metrics querying, stats, quick-stats. Health status, components, summary.

### Task 4: AlertsApiE2ETest
Alert rules CRUD, toggle, events lifecycle, providers listing/testing.

### Task 5: SloApiE2ETest + ProfilingApiE2ETest
SLO CRUD, evaluation, history. CPU profiling, memory snapshots, thread dumps.

### Task 6: SqlAnalyzerApiE2ETest + ServiceMapApiE2ETest
SQL analysis, slow queries, top queries. Service map, nodes, connections.

### Task 7: AuthenticationE2ETest + WebSocketE2ETest
Auth flows (basic, API key, session, CSRF, rate limiting). WebSocket log streaming.

### Task 8: Build verification
Run `mvn test` to verify all E2E tests pass. Fix any issues.
